## Resolution

**Same root cause as #1345 (bearing) and #1346 (mathopt3): pipeline test infrastructure, not emit/AD.** The Goddard rocket MCP regenerated by the pipeline's presolve retry is correct ÔÇö when the source's NLP pre-solve runs (an "Optimal solution. Reduced gradient less than tolerance" finding objective Ôëł 1.0128), the warm-started MCP solves cleanly.

The pipeline runner ran GAMS with `cwd=tmpdir`. After #1275 (commit `809b5008`) changed `--nlp-presolve` to emit a repo-relative `$include "data/gamslib/raw/rocket.gms"`, that include failed to resolve under tmpdir (`Error 282`). The source's NLP pre-solve never ran, no warm-start was produced, and PATH converged to `model_infeasible (status 5)` on a cold start.

### Fix

`scripts/gamslib/test_solve.py::solve_mcp` now runs GAMS with `cwd=str(PROJECT_ROOT)` (so the include resolves) and passes `ScrDir=<tmpdir>` (so scratch files stay out of the repo). See #1345 for the detailed fix description and the regression test.
