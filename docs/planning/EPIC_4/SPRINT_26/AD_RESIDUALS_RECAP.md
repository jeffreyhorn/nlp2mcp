### 2.2 #1334 fingerprint check (spurious `sum(t__, ...)` wrap)

ISSUE_1334.md §Buggy Emit specifies the pattern `sum(t__, (del(t__) * x(tt) * 0.365 * (1 - c)) * nu_kdef)`. Searching otpop emit on current main:

```bash
$ grep -E "sum\(t__" /tmp/sprint26-task7/otpop_mcp.gms
stat_p(tt).. ... + sum(t__, ((-1) * (del(t__) * x(tt) * 0.365 * (1 - c))) * nu_kdef)$(t(tt)) - piL_p(tt) =E= 0;
stat_x(tt).. ... + sum(t__, ((-1) * (del(t__) * 0.365 * (1 - c) * p(tt))) * nu_kdef)$(t(tt)) + (((-1) * (v * p(tt+(card(tt)-ord(tt))) * (-0.365))) * nu_zdef)$(t(tt)) + ... =E= 0;
```

⚠ **The #1334 spurious-sum pattern is STILL PRESENT** in current main's otpop emit. The `sum(t__, ...)` wraps the `del(t__) * x(tt)` and `del(t__) * p(tt)` cross-terms in `stat_p` and `stat_x`. Per ISSUE_1334.md §"Math vs Emit", this over-counts the `del(t__)` contribution by a factor of `|t|` (number of years).

**Despite #1334 being CLOSED on GitHub** (closed 2026-05-05): the bug pattern documented in the issue is still observable in otpop's emit on current main. Possible explanations:

1. **The closure was for a different sub-shape** — perhaps a related symptom that turned out not to be `sum(t__, ...)`.
2. **The closure was premature** — fix landed on a different code path that didn't actually eliminate the otpop case.
3. **The issue scope was reframed** mid-investigation (e.g., closed as duplicate of another issue that was then itself closed).

**Recommended Sprint 26 Priority 5 action**: re-open #1334 OR file a follow-up issue capturing the still-extant otpop emit pattern, with cross-reference to the prior closure. The Sprint 26 Day 1 closure-mechanics (Task 4 PATTERN_A_RECLASSIFICATION_PLAN.md §"Issue #1142") could include an "investigate #1334 closure" sub-step.

### 2.4 #1357 fingerprint check (`$171` domain violations in stationarity)

Per Task 4 PATTERN_A_RECLASSIFICATION_PLAN.md §"Issue #1142" and Task 3 PATTERN_C_HYPOTHESIS_VALIDATION.md §2.4, the otpop `$171` errors are NOT in stationarity equations — they're in:

Different code paths (`src/kkt/stationarity.py` for #1334 vs `src/kkt/complementarity.py` + `src/emit/emit_gams.py` bound-fixup emission for #1357). Different bug shapes (Jacobian-transpose wrap vs comp_up reference widening).

⚠ **#1334 does NOT subsume #1357.** They are independent bugs requiring independent fixes.
